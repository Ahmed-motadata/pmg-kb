# Support Portal

**Path:** Admin → Support Channels → Support Portal
(`/admin/support-channels/support-portal`)

Controls what requesters can see and do in the self-service portal. The page
description reads: "Customize the self-service portal for requesters." A **Search** box
at the top filters the settings by name; **Update** saves, **Cancel** discards.

All settings are grouped under section headings. States shown are this environment's
current values.

## Request

| Setting | State here |
|---|---|
| Allow Requester to create Incident | On |
| Allow Guest Requester to Report a Request | On |
| Allow Requester to Create Incident On Behalf Of Other Requester | Off |
| Allow Requester to View Request Due By | Off |
| Allow Requester To Access Solution | On |
| Allow Requester to Close Request | Off |
| Allow Requester To Submit Feedback | Off |
| Mandate comment to Reopen Request | Off |
| Allow Requester to Reopen Resolved Request | Off |
| Allow Requester to Reopen Closed Request | Off |
| Allow Requester to access Audit Trail | Off |

Plus **Requester Ticket Visibility** (with an info icon) — a multi-select of which
tickets a requester sees; here it holds one tag, **My Requests**.
