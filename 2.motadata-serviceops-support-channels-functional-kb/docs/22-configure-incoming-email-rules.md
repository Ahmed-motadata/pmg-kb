# Configure Incoming Email Rules — dialog

Opened by the **Configure Incoming Email Rules** button on the Incoming Email Servers
page. Applies conditions and actions to arriving mail (for example, to route or handle
specific senders differently).

## Layout

A dialog titled **Configure Incoming Email Rules** with **Add Incoming Email Rule** and
**Update** / **Cancel**. Unlike the outgoing dialog there are no module tabs — incoming
rules are a single list.

## A rule

**Add Incoming Email Rule** adds a block with:

- **Enabled** — toggle;
- **Conditions** — the same builder as outgoing rules: **Add Condition Group** creates
  a group holding a field picker, **Add Condition** adds rows, a delete button removes
  a row, and **Remove All Condition** clears everything;
- **Action** — an **Add Action** button that adds an action row (a picker for the
  action to run when the conditions match).

No rules were configured in this environment; the exact option lists inside the
condition and action pickers were not captured.
